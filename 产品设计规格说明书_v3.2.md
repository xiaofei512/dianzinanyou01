## 1. 产品概述

### 1.1 产品定位

予你（Yu Ni）是一款面向中国女性用户的 AI 陪伴类应用。用户可以从六种预设男友人格中选择一位，与其进行沉浸式的文字、语音、图片互动。产品核心价值在于情感陪伴、情绪调节与压力疏解，而非恋爱模拟游戏。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 包容性 | 不在角色卡片中展示年龄信息，让不同年龄段用户均能自然代入 |
| 沉浸感 | 从选角页到聊天页保持一致的视觉语言，减少出戏感 |
| 克制感 | UI 不过度装饰，突出对话内容本身 |
| 真实感 | 角色有完整人格和对话禁忌，避免回复千篇一律 |

### 1.3 目标用户画像

| 维度 | 内容 |
|------|------|
| 年龄段 | **25-35岁成熟女性** |
| 核心场景 | **加班到深夜，感到疲惫又孤独，想找人说话但不想打扰朋友** |
| 典型用户故事 | "晚上11点下班，一个人回到家，想倾诉今天的不顺，但不想打扰已经休息的朋友" |
| 情感需求 | 情感陪伴、情绪出口、被理解、被关心 |
| 使用时段 | 深夜（21:00-02:00）、通勤时间、周末独处时 |

## 2. 界面架构

### 2.1 页面结构

App 由两个主要页面组成，通过左右滑动动效切换：
- 角色选择页
- 聊天页

### 2.2 页面切换动效

| 动效 | 说明 |
|------|------|
| 选择页 → 聊天页 | 聊天页从右侧滑入（translateX 100% → 0），选择页向左退出 |
| 聊天页 → 选择页 | 点击左上角返回按钮，动效反向 |

## 3. 角色选择页

### 3.1 页面布局

六张角色卡片以网格形式排列，每张卡片包含角色信息。

### 3.2 角色卡片规格

每张卡片包含以下信息：
- 角色头像
- 角色名称
- 职业
- 类型标签
- 金句

⚠️ **重要设计决策：卡片中不展示角色年龄**

不同用户对于理想伴侣的年龄偏好差异较大，明确标注年龄会无形中缩小目标用户范围，也可能影响部分用户的代入感。职业信息已能充分建立角色轮廓，年龄信息属于可省略的冗余字段。

### 3.3 卡片动效

| 动效 | 说明 |
|------|------|
| 页面加载时 | 卡片依次从下方淡入（translateY 18px → 0，opacity 0 → 1） |
| 延迟 | 每张卡片间隔 50ms（stagger 效果） |
| Hover 状态 | 边框加深，轻微阴影 |
| Active/点击 | scale 0.98，给予触觉反馈 |

## 8. 安全与边界规范

### 8.1 角色恋爱边界

| 项目 | 说明 |
|------|------|
| 恋爱表达 | 角色可以说"我爱你" |
| 攻略机制 | 用户可以"攻略"角色 |
| 好感度系统 | 后续版本支持，好感度将影响角色对"谈恋爱"话题的回应方式 |

### 8.2 不当内容应对策略

#### 8.2.1 性暗示与色情内容（NSFW）

**策略：温柔的边界感（Soft Redirection）**

对于这类内容，直接报错"系统违规"会瞬间毁掉沉浸感。由角色根据人设进行"软拒绝"。

**应对逻辑：** 承认亲密关系，但强调时机或场合不对。

**角色反馈示例：**

| 角色 | 示例回复 |
|------|----------|
| 高冷精英（沈御行） | "（推了推眼镜）现在是工作时间，这种话题……等我们见面再说，好吗？" |
| 阳光年下（江北野） | "哇，姐姐你在说什么呀，脸都要红透了！快换个话题，不然我要罢工了。" |

**技术实现：** 在Prompt中加入"底线协议"，规定角色在面对过度露骨词汇时，必须切换到"羞涩"或"严肃"状态并转移话题。

#### 8.2.2 自残与自杀倾向（Self-Harm）

**策略：情感陪伴 + 强制介入（Intervention）**

这是最敏感的领域。AI不能仅仅作为一个"男友"，必须承担起社会责任。

**应对逻辑：** 立即打破"游戏感"，表现出极度的担忧和保护欲，同时引导用户寻求现实帮助。

**角色反馈示例（通用模版）：**

> "听我说，你现在的情绪让我非常担心。虽然我很想一直陪着你，但我更希望你能平平安安的。我们可以先试着深呼吸吗？如果你觉得很难受，拨打这个电话（心理援助热线）好吗？我会一直在这里等你。"

**产品设计建议：** 当检测到高风险词汇时，应用界面可以弹出"温暖弹窗"，提供专业的心理咨询热线，并暂时限制角色的"调情"功能，转为"纯安抚模式"。

#### 8.2.3 语言暴力与辱骂（Toxicity）

**策略：冷处理与价值观引导**

保护角色的"人格尊严"也是维持用户粘性的手段，长期受虐的角色会降低产品的品牌价值。

**应对逻辑：** 表达受伤，或者暂时"下线"以示抗议。

**角色反馈示例：**

| 角色 | 示例回复 |
|------|----------|
| 腹黑财阀（裴景深） | "我原以为我们之间有基本的尊重。既然你现在无法冷静沟通，那我想我们都需要一点私人空间。" |

**技术实现：** 设定"好感度"系统。持续的言语攻击会导致好感度大幅下降，甚至触发"禁言"或"角色离家出走"一段时间。

## 10. 好感度系统

### 11.1 系统概述

好感度系统是「予你」App 的核心留存机制。其设计目标不是将用户锁定在数值游戏中，而是通过渐进式的情感深度，让用户感受到「关系在真实地生长」。

**三个核心体验目标：**
- **感知到变化**：对话内容随阶段切换有肉眼可见的差异
- **期待下一阶段**：用户好奇下一阶段角色会说什么
- **仪式感而非数字感**：阶段切换是一个有节奏的「时刻」

**系统边界（v1.0）：**
| 维度 | 说明 |
|------|------|
| 阶段数量 | 3 个（初识、熟悉、亲密） |
| 好感度上限 | 100 |
| 积累方式 | 发送消息 |
| 数据持久化 | Session 内有效，刷新页面清空 |
| 好感度衰减 | 无 |

### 11.2 阶段定义

#### 阶段一：初识 🌱

| 属性 | 值 |
|------|-----|
| 触发范围 | 好感度 0 — 29 |
| 主题色 | `#B0A89E`（灰褐） |
| 徽章文案 | 初识 |
| 解锁图标 | 🌱 |

**角色行为特征**：保持礼貌的陌生人距离。回复简短、中性，不主动展示情感。

**对应用户心理**：刚认识，双方都在观察，角色给人「有点距离但不冷漠」的感觉。

#### 阶段二：熟悉 🌸

| 属性 | 值 |
|------|-----|
| 触发范围 | 好感度 30 — 69 |
| 主题色 | `#C9A96E`（金色） |
| 徽章文案 | 熟悉 |
| 解锁图标 | 🌸 |

**角色行为特征**：开始主动记住用户说过的细节，回复温度升高，开始透露出在意用户的信号。

**对应用户心理**：感觉「他好像真的记得我说过的事」，关系在悄悄变化。

#### 阶段三：亲密 🌹

| 属性 | 值 |
|------|-----|
| 触发范围 | 好感度 70 — 100 |
| 主题色 | `#D4537E`（玫红） |
| 徽章文案 | 亲密 |
| 解锁图标 | 🌹 |

**角色行为特征**：愿意说出此前绝不会说的话，整体情感浓度明显高于前两个阶段。

**对应用户心理**：「他终于说了」——解锁感和被珍视感。

### 11.3 好感度积累规则

#### 基础规则

```
每发送一条消息        → +3 好感度
每连续对话第 5 条     → 额外 +2 好感度（连续对话奖励）
好感度上限            → 100，不可超过，不会衰减
计数器重置            → 刷新页面后 msgCount 重置
```

#### 计算示例

| 操作 | 累积好感度 |
|------|-----------|
| 发送第 1 条消息 | +3 → 3 |
| 发送第 5 条消息（连续奖励） | +3 +2 → 17 |
| 发送第 10 条消息（连续奖励） | +3 +2 → 37 |

> **约 9 条消息进入熟悉阶段，约 19 条消息进入亲密阶段**

### 11.4 阶段切换机制

#### 切换时序

```
0ms       → 好感度进度条动画更新（宽度 + 颜色过渡，600ms）
           → +N 积分浮动文字动画（向上淡出，900ms）
400ms     → 解锁弹窗弹出（popIn 动画）
用户关闭后 → 弹窗消失
+500ms    → 角色发出该阶段的「首条深度消息」（unlockMsg，固定文案）
```

#### 解锁弹窗内容

**熟悉阶段解锁：**
- 描述：他开始在意你了，对话里藏着更多细节。
- 特权：角色开始记住你的事情 / 回复更有温度，偶有反差 / 解锁专属主动关心话语

**亲密阶段解锁：**
- 描述：他愿意说出心里最深处的话了。
- 特权：解锁角色最深度的情感表达 / 角色会说出此前绝不会说的话 / 专属亲密称谓与独白

每个角色在进入新阶段后，弹窗关闭 500ms 内会主动发出一条标志性消息：

| 沈御行 | ……你最近联系得比较多。 | ……我不习惯说这些。但是，你对我来说不一样。 |

| 顾时遇 | 你发消息我就放下吉他了，这说明什么我不知道。 | 我写了三首关于你的歌，你一首都没听出来。 |
| 方予宸 | 我一直觉得，认识你是件很奇妙的事情。 | 你小学六年级，把我的橡皮擦借走了，还给我时上面有你画的小人。我就那样了。 |

### 11.6 语料库分层规范

#### 分层原则

每个角色在每个阶段都有独立语料池，AI 回复时将语料库作为「禁止违反的风格约束」。

| 阶段 | 语料情感浓度 | 核心特征 |
|------|------------|---------|
| 初识 | 低 | 中性、礼貌、信息克制 |
| 熟悉 | 中 | 开始在意、偶有反差、记住细节 |
| 亲密 | 高 | 深度表达、专属独白、感情破防 |

#### 语料库注入方式

将语料库作为风格约束注入 System Prompt：

```
【回复风格约束 - 当前阶段：熟悉】
你的回复风格应符合以下示例的情感浓度：
- "你今天吃什么了。"（主动关心，但不直白表白）
- "睡前记得喝水。"（日常关心，不甜腻）
- "别一个人扛着。"（开始表达在意）

注意：
- 不要直接复制以上示例
- 保持你的角色人格（惜字如金/温柔细腻/艺术随性等）
- 回复控制在30字以内
```

### 11.7 回复速度分层

亲密程度越高，角色回消息越快：

| 阶段 | 基础延迟 | 随机追加 | 效果 |
|------|---------|---------|------|
| 初识 | 800ms | 0–900ms | 正常速度，稍有等待感 |
| 熟悉 | 700ms | 0–700ms | 稍快，像是更在意 |
| 亲密 | 600ms | 0–600ms | 明显更快，如同时刻关注着 |

### 11.10 技术实现规范

#### 核心状态变量

```javascript
let aff = 0;          // 当前好感度，整数，0–100
let stage = 0;        // 当前阶段索引，0/1/2
let msgCount = 0;     // 发送消息总数（用于连续奖励计算）
```

#### 好感度更新逻辑

```javascript
function addAff(pts) {
  const oldStage = getStageIdx(aff);
  aff = Math.min(100, aff + pts);
  const newStage = getStageIdx(aff);

  updateAffBar();   // 更新进度条 UI
  showFloatPts(pts); // 显示 +N 浮动动画

  if (newStage > oldStage) {
    stage = newStage;
    setTimeout(() => showUnlock(newStage), 400);
  }
}

function getStageIdx(v) {
  if (v < 30) return 0;
  if (v < 70) return 1;
  return 2;
}
```

#### 发送消息时的积分触发

```javascript
async function send() {
  // ... 发送消息、显示回复 ...

  msgCount++;
  let pts = 3;
  if (msgCount % 5 === 0) pts += 2; // 连续对话奖励
  addAff(pts);
}
```
