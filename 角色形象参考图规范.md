# 角色形象参考图规范

> 本文档是《产品设计规格说明书》的补充材料，定义了六个角色的官方形象参考图及其使用规范。

---

## 一、图片规范总则

| 项目 | 规范 |
|------|------|
| 图片类型 | 三视图（左侧面 + 正面 + 背面） |
| 图片格式 | PNG（建议透明或纯色背景） |
| 分辨率 | 建议 1024×1024 或更高 |
| 背景要求 | 浅灰/白色纯色背景，便于AI提取特征 |
| 人物风格 | 东亚男性，写实/半写实风格 |
| 风格统一性 | 六个角色画风需保持一致 |

## 三、AI图像生成使用规范

### 3.1 触发条件

| 场景 | 说明 |
|------|------|
| 用户主动请求 | 用户说"发张照片给我看看"、"拍张照"等 |
| 随机触发 | 每3-5轮对话内随机发一张图片 |

### 3.2 图片内容类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 自拍 | 角色视角的自拍照 | 刚下班、在咖啡店、睡前 |
| 生活照 | 角色日常生活场景 | 办公室、健身房、录音室 |
| 风景照 | 角色分享的风景 | 窗外风景、路边的花 |
| 物品照 | 角色分享的物品 | 刚买的书、做的菜、养的猫 |

### 3.3 图片消息展示规范

#### 3.3.1 气泡内图片尺寸

| 项目 | 规范 |
|------|------|
| 尺寸策略 | 自适应（根据图片比例自动调整） |
| 最大宽度 | 气泡宽度的 100%（不超过 280px） |
| 最大高度 | 320px（超出后等比缩放） |
| 最小宽度 | 120px |
| 圆角 | 与消息气泡一致（18px，左上角 4px） |
| 加载状态 | 显示骨架屏或加载动画 |

#### 3.3.2 图片预览交互

| 交互 | 说明 |
|------|------|
| 点击触发 | 点击气泡内图片，进入全屏预览模式 |
| 全屏查看 | 图片居中显示，背景半透明黑色遮罩（rgba(0,0,0,0.9)） |
| 缩放支持 | 支持双指缩放（移动端）/ 鼠标滚轮缩放（桌面端） |
| 拖动支持 | 缩放后支持拖动查看图片细节 |
| 关闭方式 | ① 点击右上角关闭按钮 ② 点击图片外区域 ③ 按 ESC 键 |
| 关闭按钮 | 白色圆形按钮，位于右上角，内含 "×" 图标 |

#### 3.3.3 加载失败处理

| 项目 | 规范 |
|------|------|
| 失败占位图 | 显示统一的"图片加载失败"占位图 |
| 占位图内容 | 灰色背景 + 断裂图片图标 + "图片加载失败"文字 |
| 重试机制 | 点击占位图可重新加载 |
| 样式参考 | 宽高与原图片占位一致，圆角保持 |

#### 3.3.4 交互流程图

```
用户点击图片
    ↓
显示全屏遮罩 + 图片居中
    ↓
┌─────────────────────────────────┐
│  [×] 关闭按钮（右上角）          │
│                                 │
│         ┌─────────┐             │
│         │         │  ← 可缩放   │
│         │  图片   │  ← 可拖动   │
│         │         │             │
│         └─────────┘             │
│                                 │
└─────────────────────────────────┘
    ↓
关闭触发：点击关闭按钮 / 点击遮罩 / 按 ESC
    ↓
退出全屏，返回聊天界面
```

#### 3.3.5 代码实现参考

```typescript
// 图片消息组件示例
interface ImageMessageProps {
  src: string;
  alt: string;
}

const ImageMessage: React.FC<ImageMessageProps> = ({ src, alt }) => {
  const [isFullscreen, setIsFullscreen] = useState(false);
  const [isLoading, setIsLoading] = useState(true);
  const [hasError, setHasError] = useState(false);

  return (
    <>
      {/* 气泡内图片 */}
      <div className="image-bubble">
        {isLoading && <Skeleton />}
        {hasError ? (
          <div className="image-error" onClick={() => retryLoad()}>
            <ImageFailIcon />
            <span>图片加载失败</span>
          </div>
        ) : (
          <img
            src={src}
            alt={alt}
            onLoad={() => setIsLoading(false)}
            onError={() => { setIsLoading(false); setHasError(true); }}
            onClick={() => setIsFullscreen(true)}
            style={{ maxWidth: '280px', maxHeight: '320px' }}
          />
        )}
      </div>

      {/* 全屏预览 */}
      {isFullscreen && (
        <ImagePreview
          src={src}
          onClose={() => setIsFullscreen(false)}
        />
      )}
    </>
  );
};
```

### 3.3 形象一致性要求

**核心原则：** 基于角色的三视图参考图生成，保持人物特征一致。

| 要求 | 说明 |
|------|------|
| 面部特征 | 五官、脸型、肤色需与参考图一致 |
| 发型 | 基本发型风格一致，可微调（如扎起来、戴帽子） |
| 服装 | 可根据场景更换，但风格需符合人设 |
| 气质 | 表情、姿态需符合角色性格 |

### 3.4 技术实现方式

```typescript
// 图像生成调用示例
const generateCharacterImage = async (
  characterId: string,
  sceneDescription: string
) => {
  // 1. 获取角色参考图
  const referenceImage = getCharacterReferenceImage(characterId);
  
  // 2. 构建prompt
  const prompt = `
    基于参考图中的人物形象，
    ${sceneDescription}，
    保持人物面部特征、发型、气质一致
  `;
  
  // 3. 调用图像生成API
  const image = await imageGenerationClient.generate({
    prompt: prompt,
    referenceImage: referenceImage,
    // ... 其他参数
  });
  
  return image;
};
```

### 5.3 语音生成流程

```typescript
// TTS 调用示例
import { TTSClient, Config } from 'coze-coding-dev-sdk';

const config = new Config();
const ttsClient = new TTSClient(config);

// 根据角色ID获取音色
const voiceMap: Record<string, string> = {
  shen: 'zh_male_m191_uranus_bigtts',
  lin: 'zh_male_ruyayichen_saturn_bigtts',
  gu: 'zh_male_taocheng_uranus_bigtts',
  fang: 'zh_male_ruyayichen_saturn_bigtts',
  jiang: 'saturn_zh_male_shuanglangshaonian_tob',
  pei: 'zh_male_dayi_saturn_bigtts',
};

const synthesizeVoice = async (characterId: string, text: string) => {
  const response = await ttsClient.synthesize({
    uid: 'user123',
    text: text,
    speaker: voiceMap[characterId],
    audioFormat: 'mp3',
    sampleRate: 24000,
  });
  
  return response.audioUri; // 返回音频URL
};
```
